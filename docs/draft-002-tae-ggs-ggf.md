## Sheet 3: การบันทึก

### แบบกลุ่ม (โรงเรียน, องค์กร, หน่วยงาน)

> ต้องมี user เข้าระบบหรือไม่? ถ้ามีก็จะลง log user ในข้อ 8

1. หน้าบันทึกให้เลือกแบบกลุ่ม
2. เลือกสาขา
3. เลือกหน่วยงานที่จะบันทึกข้อมูล
4. แสดงปฏิทินตั้งแต่วันที่เริ่มสมัครจนถึงวันนี้
   - 4.1 วันที่มีการบันทึกแล้ว → พื้นสีเขียว
   - 4.2 วันที่ผ่านไปแล้วแต่ยังไม่บันทึก → พื้นสีแดง
   - 4.3 วันที่ยังไม่ถึงแต่อยู่ช่วงร่วมโครงการ → พื้นสีเหลือง
5. เลือกวันที่บันทึกจากปฏิทิน (เลือกได้ตั้งแต่วันที่เริ่มสมัครจนถึงวันนี้)
6. ระบุจำนวนผู้ปฏิบัติของวันที่เลือก โดยระบุได้ไม่เกินจำนวนสูงสุดที่แจ้งไว้ แบ่งตามเพศ
   - สามารถระบุจำนวนตามรอบที่ปฏิบัติ เช่น เช้า 5 นาที, กลางวัน 5 นาที, เย็น 5 นาที
   - เพื่อสะท้อนผลบันทึกปฏิบัติตามจริง
7. ลงชื่อผู้บันทึกและเบอร์ติดต่อ
8. ลง system log ตาม user ที่บันทึก
9. หากเลือกบันทึกซ้ำวัน → เป็นการอัพเดตข้อมูลบันทึก (ไม่สร้างซ้ำ)

### แบบรายคน

> คาดว่าทำแบบ GGF ที่ไม่ต้องมี user เข้าระบบ

1. หน้าบันทึกให้เลือกแบบรายคน
2. เลือกสาขา (อาจส่งเป็น URL ให้แต่ละสาขาไปเลย)
3. เลือกชื่อผู้ปฏิบัติ (อาจมี feature ค้นหาด้วยชื่อ หรือออกเลขรหัสช่วยค้นหา)
4. แสดงปฏิทินตั้งแต่วันที่เริ่มสมัครจนถึงวันนี้
   - 4.1 วันที่มีการบันทึกแล้ว → พื้นสีเขียว
   - 4.2 วันที่ผ่านไปแล้วแต่ยังไม่บันทึก → พื้นสีแดง
   - 4.3 วันที่ยังไม่ถึงแต่อยู่ช่วงร่วมโครงการ → พื้นสีเหลือง
5. เลือกวันที่บันทึกจากปฏิทิน (เลือกได้ตั้งแต่วันที่เริ่มสมัครจนถึงวันนี้)
6. ติ๊กเลือกรอบที่ปฏิบัติ (บังคับเลือกอย่างน้อย 1 ช่อง):
   - เช้า 5 นาที
   - กลางวัน 5 นาที
   - เย็น 5 นาที
7. ลง system log ตามที่บันทึก
8. หากเลือกบันทึกซ้ำวัน → เป็นการอัพเดตข้อมูลบันทึก (ไม่สร้างซ้ำ)

## Sheet 4: หน้าจอ

### Home

- แสดงผลรวมนาทีที่ปฏิบัติร่วมกันทั้งหมด (และวันเวลาที่อัพเดตล่าสุด)
- มี progress เพื่อแสดง milestone ที่จะไปถึง 49 ล้านนาที
- แสดงแผนที่ประเทศไทย และขึ้นจำนวนนาทีตามภาค
- Filter ลงในระดับจังหวัด หรือสาขาได้
- รองรับภาษา TH/EN

### Dashboard

- เลือกดูข้อมูลตามมุมมองที่ต้องการ เช่น เพศ, ช่วงอายุ (เฉพาะรายคน), รายจังหวัด

### Report

- ดึงรายงานผลการปฏิบัติโดยดึงตามรายสาขา
  - แบบกลุ่ม
  - รายคน
