# แนวทางตรวจจับความผิดปกติ (Anti-fraud / Anti-spam)

> เอกสารประกอบการประชุม — เสนอแนวทางป้องกันการปั๊มยอดนาที
>
> บทเรียน: โครงการสวดลักขี เคยพบปัญหาผู้ใช้บันทึกยอดสูงผิดปกติ เนื่องจากไม่ได้ตีกรอบการบันทึกไว้ดีพอ

---

## ระดับ 1: Rule-based (ทำได้ทันที)

กฎพื้นฐานที่ตั้งไว้ในระบบตั้งแต่วันแรก

| กฎ | ค่า | เหตุผล |
|---|-----|--------|
| **เพดานต่อครั้ง** | 5 นาที | วิทิสาสมาธิครั้งละ 5 นาที |
| **เพดานต่อวันต่อคน** | 15 นาที | 3 ครั้ง/วัน (เช้า กลางวัน เย็น) × 5 นาที |
| **เพดานต่อองค์กรต่อวัน** | จำนวนคน × 5 นาที (เช่น 500 คน × 5 = 2,500 นาที/วัน) |
| **Cooldown** | **ยังไม่ยืนยัน** — ปิดไว้ก่อน, รอหารือในที่ประชุม |

> วิทิสาสมาธิ = รูปแบบการปฏิบัติสมาธิ ครั้งละ 5 นาที สูงสุด 3 ครั้ง/วัน (เช้า กลางวัน เย็น)

## ระดับ 2: Pattern Detection (ตรวจจับพฤติกรรมแปลก)

ระบบวิเคราะห์ข้อมูลย้อนหลัง flag รายการที่น่าสงสัย

| รูปแบบ | วิธีตรวจ |
|--------|---------|
| **บันทึกซ้ำเป็นจังหวะ** | ทุก 5 นาทีตลอดทั้งวัน → น่าจะเป็น bot/script |
| **ยอดกลมเกินไป** | องค์กรบันทึก 10,000, 20,000 ทุกวันเป๊ะ → น่าสงสัย |
| **เวลาผิดปกติ** | บันทึกตี 2-5 ทุกคืน ต่อเนื่องหลายวัน |
| **IP/Device ซ้ำ** | หลาย "คน" บันทึกจาก device เดียวกันรัวๆ |
| **ยอดพุ่งฉับพลัน** | สาขาที่เคยบันทึก 1,000 นาที/วัน อยู่ดีๆ กลายเป็น 50,000 |

## ระดับ 3: สาขาเป็นด่านตรวจ (Human Verification)

ใช้คนที่อยู่ในพื้นที่เป็นผู้ตรวจสอบความถูกต้อง

| กลไก | รายละเอียด |
|------|-----------|
| **สาขา approve ยอดรายวัน** | ยอดองค์กร/โรงเรียน ต้องให้สาขากดยืนยันก่อนเข้าระบบ |
| **Flag → Review** | ระบบ flag รายการที่ผิดปกติ → สาขาตรวจสอบก่อนนับ |
| **แนบหลักฐาน** | องค์กร/โรงเรียนแนบรูปกิจกรรม สุ่มตรวจได้ |
| **Dashboard สาขา** | สาขาเห็นภาพรวมสมาชิกตัวเอง สังเกตเห็นยอดแปลกๆ ได้เร็ว |

---

## สรุป: แนะนำลำดับการทำ

```
เปิดระบบวันแรก          มีข้อมูลจริงแล้ว          ระยะยาว
──────────────         ──────────────          ─────────
ระดับ 1: Rule-based    ระดับ 2: Pattern         ปรับค่า threshold
ระดับ 3: สาขาตรวจ     Detection                ตามข้อมูลจริง
```

**เริ่มจากระดับ 1 + 3 ก่อน** — ทำง่าย ได้ผลทันที ส่วนระดับ 2 ค่อยเพิ่มเมื่อมีข้อมูลจริงแล้ว
